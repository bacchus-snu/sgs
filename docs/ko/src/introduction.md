# SNUCSE GPU 서비스 3.0

SNUCSE GPU 서비스(SGS) 3.0는 서울대학교 컴퓨터공학부 학생들에게 무료로 제공되는 서비스입니다.
서비스를 사용하기 전에 먼저 이 매뉴얼을 읽어주세요.

이 서비스는 **연구 목적으로만** 제공됩니다.
워크스페이스 신청서에 적은 용도로만 사용해주세요.

## 서비스 운영 방침

바쿠스 구성원들이 최선을 다해 서비스를 제공하고 있습니다.
모든 사용자에게 원활한 서비스를 제공하기 위해 노력하고 있지만, 기술적 문제, 유지보수, 다른 사용자의 자원 사용 등으로 인해 서비스가 중단될 수 있습니다.

컨테이너는 언제든 경고 없이 종료될 수 있습니다.
데이터 손실을 방지하기 위해 중요한 데이터는 반드시 지속 볼륨에 저장해주세요.
종료된 컨테이너의 데이터는 복구할 수 없습니다.

## 자원 할당 정책

자원 할당량에는 두 가지 종류가 있습니다: 보장 할당량과 한도 할당량.

보장 할당량은 Pod가 실행되는 동안 독점적으로 사용할 수 있는 자원입니다.
즉, 컨테이너가 아무 작업도 하지 않더라도 다른 사용자는 해당 자원을 사용할 수 없습니다.

보장 할당량(GPU 포함)을 과도하게 할당했는데 아무 작업도 하지 않는 경우, 다음과 같은 조치를 취할 수 있습니다:

- 해당 컨테이너 종료
- 워크스페이스 비활성화
- 영구적인 서비스 이용 제한

한도 할당량은 워크스페이스 내 모든 컨테이너가 사용할 수 있는 자원의 최대량입니다.
여유있게 신청할 수 있지만, 보장 할당량을 설정하지 않으면 자원 제공이 보장되지 않습니다.

## 정리 작업

바쿠스는 자원 회수를 위해 오래된 워크스페이스를 주기적으로 정리합니다.
정리 작업은 다음 시기에 진행됩니다:

- 봄/가을학기 시작 시
- 여름/겨울방학 시작 시

정리 작업 전에 워크스페이스 사용자에게 연락드립니다.
기한 내에 응답이 없으면 워크스페이스가 먼저 비활성화되고, 이후 삭제됩니다.
한 번 삭제된 워크스페이스의 데이터는 복구할 수 없습니다.

## 보안 정책

각 워크스페이스는 여러 사용자가 함께 사용하는 리눅스 시스템과 비슷한 보안 수준을 가집니다.

예를 들어, 다음 정보는 다른 사용자도 볼 수 있습니다:

- Pod와 컨테이너의 이름
- 컨테이너의 이미지
- 컨테이너에서 실행 중인 명령어

반면, 다음 정보는 다른 사용자가 볼 수 없습니다:

- 지속 볼륨과 임시 저장소의 내용
- Secret의 내용
- 컨테이너의 로그

서비스의 보안을 테스트해야 하는 경우, 미리 바쿠스에 연락해주세요.
