# 워크스페이스 신청

[id]: https://id.snucse.org
[sgs-request]: https://sgs.snucse.org/request

> 시작하기 전에 `undergraduate` 또는 `graduate` 그룹에 등록된 [스누씨 계정][id]이 필요합니다.

워크스페이스를 신청하려면 SGS [워크스페이스 관리 페이지][sgs-request]에서 신청 양식을 작성하세요.

![워크스페이스 신청 양식](images/request-workspace/ws-request.png)

양식을 제출하면 워크스페이스가 "승인 대기" 상태가 됩니다.

![승인 대기](images/request-workspace/ws-pending.png)

바쿠스 부원이 워크스페이스 신청을 검토합니다.
승인되면 워크스페이스가 "사용 가능" 상태로 전환됩니다.

![사용 가능](images/request-workspace/ws-enabled.png)

사용자 목록이나 할당량 업데이트는 언제든지 요청할 수 있습니다.
"이유" 필드에 변경 목적을 설명하세요.
워크스페이스 신청과 마찬가지로 변경 요청도 바쿠스 부원이 검토합니다.

![변경 요청](images/request-workspace/ws-changes.png)

## GPU 자원 할당 요청 가이드라인

**중요:** 제한된 GPU 자원이 최적이고 투명하게 할당될 수 있도록, 모든 연구자에게 자원 요청에 대한 명확하고 간결한 설명을 요청합니다.

GPU 자원을 요청할 때는 할당된 자원을 연구 목표 달성을 위해 어떻게 활용할 것인지 설명해주세요.
다음 내용이 포함되어야 합니다:

- `graduate` 노드그룹에서 워크스페이스를 요청하는 경우, 지도교수 이름 및/또는 프로젝트 PI(Principal Investigator) 이름을 반드시 포함하세요.
- 목표와 목적을 포함한 연구 프로젝트의 간략한 개요
- 특정 저널이나 학회를 목표로 하는 경우, CfP(Call for Papers) URL을 포함하세요.
  - 마감일이 가까운 경우 자원 요청의 우선순위를 정하는 데 도움이 됩니다.
- 필요한 GPU 자원에 대한 설명:
  - GPU에서 실행할 워크로드에 대한 간략한 설명
  - 여러 GPU를 요청하는 경우, VRAM 요구사항이나 연구 과제와 관련하여 상세히 설명하세요.
- 저장소 요구사항에 대한 설명. 다음 세부정보를 포함하여 특정 양의 저장소가 필요한 이유를 설명하세요:
  - 모델의 크기 (예: 모델 파라미터 수)
  - 데이터셋의 크기 및 개수
  - 특정 저장소 관련 제약 사항이나 한계
- 'CPU 보장' 또는 '메모리 보장'에 0이 아닌 값을 입력한 경우, 요청에 대한 상세한 설명을 제공하세요.
  - 대부분의 경우 보장된 CPU 또는 메모리 자원이 필요하지 않습니다. 보장된 자원은 미활용 시 워크로드가 종료될 위험만 높입니다.
- 프로젝트의 예상 기간

<div class="warning">
설명이 불충분하면 자원 요청이 지연되거나 거부될 수 있습니다.
</div>

<div class="warning">
여러 연구자가 참여하는 공동 프로젝트의 경우, 단일 워크스페이스를 신청한 후 id.snucse.org 계정 사용자 이름을 제공하여 팀원을 추가하세요.
각 연구자가 개별 워크스페이스를 요청할 필요가 없습니다.
</div>

신청서는 한국어 또는 영어로 제출할 수 있으며, 검토자는 두 언어 모두로 평가할 수 있습니다.

### 왜 중요한가요?

시스템 관리자로서 우리는 GPU 자원이 연구 프로젝트에 최대한 도움이 되도록 할당할 책임이 있습니다.
자원 요청에 대한 명확한 설명을 제공하면 다음에 도움이 됩니다:

- 각 요청의 가치와 긴급성을 평가하고 우선순위를 정합니다.
- GPU 자원의 효과적인 활용과 연구 결과에 미치는 영향을 컴퓨터공학부에 보고합니다.
- 연구 커뮤니티를 더 잘 지원하기 위해 자원 할당 프로세스를 지속적으로 개선합니다.

이해해 주셔서 감사드리며, [contact@bacchus.snucse.org](mailto:contact@bacchus.snucse.org)로 언제든지 문의하세요.
