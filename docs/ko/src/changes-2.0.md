# 2.0 변경사항

SGS 2.0을 사용해보셨다면, 3.0에서 달라진 점을 확인해주세요.

## 워크스페이스 신청

워크스페이스와 할당량을 관리할 수 있는 통합 시스템을 개발했습니다.
이제 웹 인터페이스에서 바로 워크스페이스를 신청할 수 있습니다.

더 이상 자원 할당량을 설정하기 위해 UserBootStrap을 작성할 필요가 없습니다.

## 노드

하나의 클러스터에서 여러 개의 노드를 관리합니다.
각 노드는 노드그룹(학부, 대학원)에 속하며, 워크스페이스의 Pod도 노드그룹에 속합니다.
워크스페이스의 Pod는 기본적으로 같은 노드그룹 내의 노드에서만 실행됩니다.
특정 노드에서 실행해야 하는 경우, Pod 설정에서 nodeSelector를 사용하세요.

## 자원 모델

CPU와 메모리의 보장 할당량은 더 이상 기본으로 제공되지 않습니다.
꼭 필요한 경우, 워크스페이스 신청서에 정당한 사유를 작성해주세요.

Pod의 CPU, 메모리 제한은 이제 LimitRange를 통해 자동으로 설정됩니다.
여러 컨테이너(하나의 Pod 내 여러 컨테이너, 여러 개의 Pod, 또는 둘 다)를 동시에 실행해야 하는 경우, Pod 설정의 제한을 조절하세요.

## 권한

노드 정보를 직접 조회할 수 있습니다.
더 이상 사용 가능한 노드의 자원을 확인하기 위해 바쿠스에 문의할 필요가 없습니다.

하나의 워크스페이스에 여러 사용자를 추가할 수 있습니다.
학교 수업 등 다른 사용자와 협업하는 경우, 워크스페이스를 공유할 수 있습니다.

기본적인 Pod 보안 표준이 적용됩니다.
제한이 너무 엄격하다고 느껴지면 문의해주세요.
